Missing Dependencies
This the most comon mistake and your effect use your updated data with realling it


old
const [count, setCount] = useState();

useEffect(() => {
    console.log("Count is", count)
},[]) count is missing

we are using count inside the effect. ut it's not in the dependency array

React seen an empty array and says "Run this once never again. so the first run count = 0 Efect captures this value and freezes.
so the user click btn 5 time and count is 5 but inside the efect is still think count as zero so it stuck with a stale snapshot
and this is called a stale closure.

A classic javascript proble that show up constantly in React. The effect captured count at the time it ran and never updated

The fix issimple 
include every value your effect uses in the dependency array
useEffect(() => {
  console.log("Count is", count)
}, [count])

# Not Handlin Async Correctlty
You'll hit this the first time you fetch data with useEffect.


useEffect( async () => {
  const res =  await fetch("/api/users);
  const data = await res.json();
  setUser(data)
},[])

And this is a problem, because useEffect  expects th cleanup to be a function or nothing. so when mark it, it return token not  function.


To fix
useEffect(() => {
  async function fetchUser() {
    const res =  await fetch("/api/users);
    const data = await res.json();
    setUser(data);
  }

  fetchUser()
},[])
